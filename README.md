# DPGMtool
This tool is designed to calculate the genetic distance at the single-gene level and correlate it with heterosis, thereby enabling the identification of candidate genes associated with heterosis.
Introduction
Using parental resequencing data, this framework systematically evaluates the correlation between genetic divergence of each gene and pheotype (for instance, better-parent heterosis, BPH) by performing variant filtering, genetic distance calculation, and haplotype analysis for all genome-wide genes individually.The detailed pipeline(Figure 1) is as follows:
1) First, heterozygous and missing genotype sites are removed from the variant call files generated by resequencing to eliminate interference from sequencing errors.
2) Next, variant annotation is performed on the filtered genome-wide variant call files to facilitate the examination of variant types at each locus.
3) After completing the aforementioned single-gene analytical procedures, genetic distance results of all genes are integrated. A t-test is then performed to identify genes with significant differences in genetic distance between groups (P < 0.05). Pearson’s correlation analysis is conducted between the genetic distance of these genes and pheotype BPH  to obtain correlation coefficients for each gene in different groups.
4) Finally, candidate genes with top 5% correlation in the experimental group and non-top 5% correlation in the control group are selected as candidate functional heterosis genes for which phenotype you wanna to analysis.
The entire pipeline mainly consists of 4 shell scripts and 2 R scripts.The shell scripts include: Dis_candidate_part1.sh, Dis_candidate_part2.sh, Dis_candidate_part3.sh, and Dis_F1_hap.sh.The R scripts include: hap.v1.r and pearsonr.r.The detailed functions of each script are as follows:
   1)Dis_candidate_part1.sh: This script is used to remove heterozygous and missing genotype sites from the genome-wide VCF file generated by variant calling, followed by variant annotation for the processed VCF. Based on the annotated genome-wide VCF file, the script performs batch analysis by calling Dis_F1_hap.sh. The function of Dis_F1_hap.sh is to extract the gene interval and 2 kb upstream sequence for each gene from the annotated VCF, thereby generating a single-gene VCF file. Subsequently, synonymous mutations and intron variants are filtered out from the single-gene VCF. Genetic distance calculation is performed using the filtered single-gene VCF. Next, the variant information of samples in the single-gene VCF is combined to generate the F₁ VCF file and the F₁ genetic distance file. Finally, hap.v1.r is used to conduct haplotype analysis for the F₁ VCF file of each gene.
   2)Dis_candidate_part2.sh: This script merges the single-gene F₁ genetic distance files generated in the previous step. After merging, pearsonr.r is used to perform correlation analysis between genetic distance and phenotypic BPH.
   3)Dis_candidate_part3.sh: This script counts the number of variant sites for each gene and calculates the number of special sites. For example, IF You wanna focus on the genetic basis of heterosis contributed by the restorer line Pos. Therefore, a special site was defined as a locus where the genotype of Pos was different from that of all other parents (for example, Pos carries genotype T, while all other parents carry genotype A).
##Usage
1)Dis_candidate_part1.sh: Input parameters include the working path for storing files, data path for built-in files, script path, software path for required tools, name of the genome-wide VCF file, interval range for extracting single-gene VCF files, gene list for analysis, prefixes for a series of output files, name of the VCF file for batch haplotype analysis, width and length for haplotype visualization, number of threads for parallel running, and an optional parameter for whether to process the genome-wide VCF file.
2)Dis_candidate_part2.sh: Input parameters include the working path, data path for built-in files, script path, interval range for single-gene VCF extraction, phenotypic file for association with genetic distance, phenotypic grouping file, genetic distance grouping file, year information, output file prefix, and two optional parameters for genetic distance processing.
3)Dis_candidate_part3.sh: Input parameters include the working path, data path for built-in files, script path, gene ID, output file prefix, and width and length for haplotype visualization.
In summary, the construction of this computational framework enables comprehensive scanning and evaluation of the potential heterotic effects of individual genes at the genome-wide level, thereby facilitating the identification of key heterosis-related genes. Moreover, this framework can greatly reduce population sequencing costs and improve the efficiency of mining heterosis genes.
